# 参考资料与引用依据

本文件用于实验报告和公开仓库中的引用整理。引用口径以工程安全部署和风险背景为主，不把本项目表述为新的安全研究方法。所有在线资料的访问日期按 `2026-06-01` 记录。

## 引用边界

- NIST Zero Trust 用于支撑“不因局域网或网络位置默认信任”的访问控制思想；本项目不声称完整实现企业级 Zero Trust Architecture。
- OpenSSH、UFW、Tailscale、Cockpit、systemd 和 Fail2Ban 文档用于支撑配置项选择、状态核查命令和部署方式。
- OWASP、AgentDojo、InjecAgent 和间接提示注入论文用于说明 Agent/Gateway/工具调用场景存在系统性风险；本项目只验证部署边界和访问控制，不声称彻底解决提示注入。
- OpenClaw 文档用于支撑 Gateway 远程访问、loopback、SSH tunnel、Tailscale 等工程选型；涉及具体能力时，以本机实际版本 `OpenClaw 2026.5.28 (e932160)` 的命令输出和审计结果为准。

## 项目表述边界

本项目表述为：

> 本项目围绕自托管大模型智能体应用，设计并验证了基于 OpenSSH、UFW、Tailscale、Cockpit、OpenClaw Gateway 和 systemd 的最小暴露面部署方案。项目通过有效配置核查、端口监听检查、成功访问与失败访问测试，证明管理面和 Agent Gateway 未直接暴露到不可信网络。

本项目不声明：

> 本项目提出了一种新的 Agent 安全框架，解决了大模型智能体的提示注入和工具越权问题。
